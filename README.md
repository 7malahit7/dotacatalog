# Dota Relics Catalog

Простой CRUD-каталог предметов Dota 2 с публичной страницей каталога, детальными страницами предметов и открытой админкой для управления записями.

## Стек

- C++17
- HTML
- CSS
- JavaScript
- SQLite
- CMake

## Требования для Linux

Проект рассчитан на запуск в Linux. Для Debian/Ubuntu-подобных систем можно установить базовые зависимости так:

```bash
sudo apt update
sudo apt install build-essential cmake sqlite3 libsqlite3-dev
```

Пакеты нужны для:

- `build-essential` - компилятор C++ и базовые инструменты сборки.
- `cmake` - генерация файлов сборки.
- `sqlite3` и `libsqlite3-dev` - локальная база данных и заголовочные файлы для подключения SQLite из C++.

## План разработки

1. Сверстать статичную главную страницу каталога.
2. Добавить статичную страницу предмета.
3. Добавить статичные страницы админки.
4. Оформить интерфейс через CSS.
5. Добавить минимальный JavaScript для формы и подтверждения удаления.
6. Реализовать HTTP-сервер на C++.
7. Подключить SQLite и хранение предметов.
8. Реализовать CRUD-операции в админке.

## Запуск прототипов

Статичные HTML-прототипы можно открыть через расширение VS Code Live Server:

1. Открыть в VS Code папку проекта `maximaster`.
2. Открыть файл `prototype/index.html`.
3. Нажать `Go Live` или выбрать `Open with Live Server`.

## Запуск C++ сервера

В проекте есть простой HTTP-сервер на C++, который отдаёт текущие HTML-прототипы и файлы из папки `static`.

Сборка и запуск:

```bash
cmake -S . -B build
cmake --build build
./build/dota_relics_catalog
```

Также можно перейти в папку сборки и запустить бинарник оттуда:

```bash
cd build
./dota_relics_catalog
```

После запуска сайт будет доступен по адресу:

```text
http://127.0.0.1:8080
```

Основные страницы:

- `http://127.0.0.1:8080/` - каталог предметов.
- `http://127.0.0.1:8080/items/1` - детальная страница предмета.
- `http://127.0.0.1:8080/admin` - админка.
- `http://127.0.0.1:8080/admin/new` - форма добавления предмета.
